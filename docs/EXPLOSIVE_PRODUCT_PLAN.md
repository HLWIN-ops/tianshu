# 天枢《爆款改造总方案》

> 版本：1.0
> 日期：2026-07-16
> 适用项目：天枢八字排盘（Cloudflare Pages 免费版、纯静态前端）
> 文档性质：阶段 3 总方案 + 阶段 4 执行基线 + 阶段 5 验收口径

## 0. 执行摘要

天枢已经具备专业排盘工具的主要骨架：公历/农历、精确时分/时辰、349 个离线城市、真太阳时、两种子时规则、四柱十神藏干神煞五行、大运流年、长篇解读、本地档案、古今人物相似度和分享海报。它当前最缺的不是继续堆术语，而是把这些能力组织成一个用户能在 30 秒内理解、愿意保存、愿意再次打开、也敢于分享的完整产品。

本轮改造的北极星方向是：

> **本地隐私优先、依据可追溯，把传统命理转译为可验证行动的个人节奏工具。**

对外的一句话差异化表达：

> **比专业排盘更易懂，比 AI 命书更透明，比大众命理更克制，比云端产品更保护隐私。**

产品不承诺“预测必然发生”，而是回答四个更可信的问题：

1. 我的出生信息经过哪些规则后得到这张盘？
2. 哪些结论直接来自盘面，哪些只是需要验证的解释？
3. 我现在最值得做的一件小事是什么？
4. 如何在不暴露出生隐私的情况下保存、比较和分享？

### 成功定义

改造成功不是“页面更丰富”，而是同时满足以下条件：

| 维度 | 目标状态 | 验收标准 |
|---|---|---|
| 首次价值 | 用户快速看到专属结论 | 新用户从打开到首个结果的中位时长不高于 60 秒；快速路径必填项不超过 5 组 |
| 可解释性 | 结果能回到盘面证据 | 首屏每条核心判断都能展开看到盘面依据、适用范围与不确定性 |
| 可用性 | 所有核心操作真实可点 | 桌面端与 360px 移动端 E2E 覆盖开场、排盘、切页、保存/恢复、分享海报，0 个未处理页面错误 |
| 留存 | 一次性报告变成周期工具 | 有最近档案恢复入口与本地月度回看；回访用户无需重复录入即可恢复结果 |
| 传播 | 分享形成可回流闭环 | 默认脱敏分享；文本与海报带运行时站点地址/CTA，不出现完整生日、时刻、地点 |
| 信任 | 风险、边界、隐私说清楚 | 隐私页、方法页、数据丢失提醒、算法范围、非医疗/非投资边界均可从产品内到达 |
| 工程质量 | 发布不再依赖人工“感觉” | 现有单元/冒烟测试与真实浏览器测试全部通过；GitHub Actions 阻止失败版本发布 |
| 商业验证 | 先验证价值，再接交易 | 在没有后端和支付能力前不伪造会员；付费只作为可测量的意向假设，不锁定基础可信能力 |

## 1. 硬约束、原则与非目标

### 1.1 硬约束

- 部署在 Cloudflare Pages 免费版，当前交付物必须是纯 HTML/CSS/JavaScript 静态文件。
- 核心排盘、解读、档案和海报在浏览器本地运行；不得假装存在服务器账户、云同步、远程 AI 或支付鉴权。
- 不采集姓名、出生时间、出生地点和命盘结果到自有服务器。
- Cloudflare Web Analytics 只有在站点控制台实际开启后才算启用；代码或文档不得宣称尚未验证的线上配置已生效。
- 未来付费、加密同步、专家服务均需新的后端、安全、合规和客服条件，不能通过前端布尔值实现“会员”。

### 1.2 产品原则

1. **先结果，后细节**：先给一眼能懂的结论，再允许用户逐层查看术语与推导。
2. **证据与解释分层**：盘面事实、规则推导、行动建议必须在视觉和文案上可区分。
3. **默认保护用户**：本地优先、分享脱敏、导入限量、风险不夸大。
4. **不贩卖焦虑**：不用灾祸、疾病、破财、感情失败等确定性恐吓推动付费或分享。
5. **专业选项渐进披露**：默认流程服务初学者，高级规则随时可展开且不会消失。
6. **每次发布可验证**：真实浏览器、移动端、键盘、失败恢复和隐私检查是发布门槛。

### 1.3 本阶段非目标

- 不实现账号、短信登录、跨设备云同步、在线支付、自动续费或远程 AI 对话。
- 不宣称医学、法律、投资或婚恋结果的确定性。
- 不用虚构的用户数、准确率、案例库或转化率做宣传。
- 不为了增长把完整出生信息写进 URL、分享文本、埋点事件或第三方脚本。

## 3. 目标人群与核心 JTBD

### 3.1 优先人群

| 人群 | 典型场景 | 真正需求 | 产品承诺 | 不做的事 | 验收标准 |
|---|---|---|---|---|---|
| 学习型初学者 | 第一次接触八字，不懂术语 | 白话理解、知道结论从哪里来 | 先给结论，再给依据和术语解释 | 用术语密度制造“专业感” | 不阅读帮助也能说出首屏结论与一条行动建议 |
| 事件驱动型自我探索者 | 职业、关系、生活节奏不确定 | 获得克制、可执行的观察框架 | 给当期主题、行动和复盘问题 | 给必然吉凶、制造恐惧 | 所有建议均是可撤销、低风险动作，并带“请结合现实验证” |
| 专业/半专业用户 | 高频排盘、管理多人档案 | 精细规则、效率、备份迁移 | 高级规则不缩水，本地档案可导入导出 | 把基础数据恢复作为付费墙 | 可管理多个档案；导入导出有版本/异常反馈 |
| 社交比较者 | 想分享人物相似度或年度主题 | 有趣、好看、不泄露隐私 | 分享身份感和行动，不分享完整生辰 | 默认生成可反推身份的卡片 | 默认海报与文案不含完整日期、时刻、城市 |
| 高焦虑/脆弱用户 | 希望工具替自己做重大决定 | 稳定感与边界 | 提醒寻求现实证据和专业帮助 | 健康、投资、法律、婚姻的确定性决策 | 风险主题出现明确边界提示，无恐惧付费 CTA |

### 3.2 核心 JTBD

用户雇佣天枢完成的任务，按优先级排列：

1. **可信排盘**：在城市、时区、真太阳时和子时规则不出错的前提下，生成一张可核对的命盘。
2. **白话行动**：把复杂盘面转成当下能执行、一个月后能复盘的建议。
3. **验证依据**：让我知道结论为何出现、可能在哪些条件下失效，而不是接受万能文案。
4. **私密管理**：安全地保存本人或家人的多个档案，并能备份、迁移、恢复。
5. **脱敏表达**：分享“我是谁/我该做什么”的卡片，同时避免泄露出生隐私。
6. **购买判断**：先验证免费价值，再决定是否需要更深主题内容；权益、价格、交付边界必须清楚。

### 3.3 价值主张验收

- 首页首屏同时出现“本地计算/隐私”“依据可追溯”“行动建议”三类信息，且不超过两屏即可理解。
- 让 5 名未使用过产品的目标用户完成首测，至少 4 名能用自己的话说出“它与普通排盘/AI 命书的不同”。
- 任意核心结论都能定位到一个或多个实际盘面变量；相同关注方向但不同盘面不应只返回同一段模板。

## 4. 产品功能矩阵

“MVP”指本轮静态站达到可公开传播的最低完整产品；“完整版”仍优先指可在纯静态边界内实现的成熟形态。“条件型未来”必须在后端、支付和合规条件齐备后另立项目。

| 能力域 | 当前基础 | MVP（P0） | 完整静态版（P1–P3） | 条件型未来 | 验收标准 |
|---|---|---|---|---|---|
| 快速排盘 | 输入项完整，高级项较多 | 必填优先，高级规则折叠；字段级报错并聚焦 | 新手/专业模式记忆；边界输入引导 | 无 | 新用户 60 秒内出结果；错误不用滚动寻找 |
| 专业规则 | 真太阳时、时区、两种子时规则 | 保留全部能力，显示校正前后和边界风险 | 规则对比沙盒、结果差异高亮 | 权威历法服务交叉验证 | 切换关键规则后，受影响柱位和原因可见 |
| Aha 摘要 | 有结果和长篇解读 | 一句身份、三条盘面事实、一条当前行动、一条不确定性 | 多主题行动卡、月度复盘对照 | 经用户授权的云端长期记忆 | 首屏无需看长文即可获得一条专属价值 |
| 依据追溯 | 有稳定性/边界信息 | “柱位稳定度/时间确定度”命名；结论可展开看依据 | 方法词典、规则版本、对比解释 | 第三方专家审校记录 | 事实、解释、建议有清晰标签，不混写 |
| 解读阅读 | 长篇内容完整 | 摘要优先、章节折叠、保留章节定位 | 搜索、收藏、本地阅读进度 | 远程生成的个性化深报 | 移动端不再要求连续滚动完整长文才能找到重点 |
| 本地档案 | localStorage 保存、导入导出 | 最近档案恢复；明示明文/清缓存丢失；导入校验、限量 | 可选本地口令加密、版本迁移、冲突预览 | 端到端加密同步 | 损坏/超限文件不能覆盖现有档案；导出后可恢复 |
| 趣味相似度 | 60 人人物库与海报 | 海报/文本默认脱敏并带站点 URL/CTA | 多卡模板、用户可控展示项 | 经授权的社区画廊 | 默认产物无完整生辰地点，扫码/地址可回站 |
| 回访机制 | 用户需自行重新打开 | 最近档案、7 天试做结论 | 本地历史对比、年度时间轴、可导出复盘 | 通知/邮件需独立同意与后端 | 关闭页面再打开，一次点击恢复最近结果 |
| 无障碍与韧性 | 基础 HTML 结构 | 键盘/焦点/ARIA；启动异常恢复面板 | 高对比、减少动画、字体缩放专项 | 无 | 键盘可走完主流程；脚本异常时不是空白页 |
| 数据分析 | 无漏斗 | 先定义隐私安全事件；仅在控制台实际开启 Cloudflare Web Analytics 后使用 | 本地实验分组 + 只上报非敏感事件 | 自建匿名事件服务 | 事件不含姓名、生辰、地点、四柱或自由文本 |
| 商业化 | 无 | 只展示透明的“需求登记/内容预览”，不限制基础能力 | 可下载的一次性内容样品、专业版需求测试 | 正式支付、授权、退款、客服、加密同步 | 无后端时无法通过改前端状态获得虚构“会员权益” |

## 5. 核心用户流程与信息架构

### 5.1 核心闭环

```text
传播卡片 / 搜索 / 直接访问
            ↓
   价值承诺 + 隐私说明
            ↓
快速输入（高级规则按需展开）
            ↓
30 秒 Aha：身份句 + 3 个证据 + 1 个行动 + 不确定性
        ↙           ↘
  查看依据/长解读       保存到本地
        ↓               ↓
  脱敏分享卡 ← 月度回看/最近档案恢复
        ↓
  运行时站点 URL 带来下一位用户
```

主链验收：新用户不进入设置、不阅读教程也可完成排盘；回访用户不重新输入即可恢复；分享接收者能通过地址回到产品入口。

### 5.2 首次使用动线

1. **入口**：一句定位、3 个信任点、“开始排盘”主按钮；隐私与方法为次级入口。
2. **快速输入**：历法、出生日期、时间、地点、性别/起运所需项；“真太阳时/子时/时区”等放入高级区。
3. **即时校验**：错误在字段旁说明，并把焦点移到第一个错误；不得只用全局 alert。
4. **结果首屏**：先看到能理解的摘要，再看到四柱表和技术信息。
5. **深入阅读**：按“盘面依据—性格/能力—关系—事业—节奏—行动”折叠展示。
6. **投入**：保存档案或完成一次 7 天试做，留下可回看的事实结论。
7. **传播**：用户自主选择分享“人物相似度”“真太阳时换柱”“今年一件事”，默认脱敏。

验收标准：主按钮在 320/360/390px 视口内始终可见可点；无被开场层或透明元素遮挡；每屏最多一个视觉主 CTA。

### 5.3 回访动线

1. 首页展示“继续查看最近档案”，同时保留“新建排盘”。
2. 恢复后直接进入结果摘要，并显示上次保存时间。
3. 用户可查看 7 天试做、更新事实结论或切换其他档案。
4. 若 localStorage 被清空，页面解释原因并提供导入入口，而不是显示故障空态。

验收标准：有档案时一次点击恢复；无档案时不展示误导性的“继续”；恢复内容与保存时输入一致。

### 5.4 信息架构

```text
首页 / 排盘
├─ 快速输入
│  └─ 高级设置：真太阳时、时区、子时规则、经度
├─ 结果
│  ├─ 今日先看：身份句、三证据、一行动、不确定性
│  ├─ 命盘：四柱、十神、藏干、神煞、五行
│  ├─ 节奏：大运、流年、当期信号
│  ├─ 解读：摘要 + 可折叠章节
│  ├─ 趣味：古今人物相似度
│  └─ 分享：脱敏预览、海报、复制文本
├─ 档案
│  ├─ 最近使用
│  ├─ 本地档案列表
│  └─ 导入 / 导出 / 数据风险说明
├─ 方法与依据
└─ 隐私与使用边界
```

导航验收：未生成结果时，“命盘/节奏/解读/趣味”呈明确的锁定/禁用状态并说明“先完成排盘”，不表现为可点击后再弹错；生成后标签可由鼠标、触摸和键盘操作，选中状态由 `aria-selected` 等语义同步表达。

## 6. 关键页面与交互规范

### 6.1 首页/输入区

- 首屏标题聚焦价值，不以功能数量开场。
- 快速表单只显示完成排盘所需字段；专业能力通过 `<details>` 或等价渐进组件展开。
- 城市选择提供搜索/建议，但必须允许用户确认当前城市与经度来源。
- 真太阳时开启时，结果中显示“钟表时 → 校正时 → 差值”，临近边界时突出核对建议。
- 表单错误采用“字段边框 + 明确原因 + 修复建议”，并保持用户已输入内容。

验收：输入非法日期、未来日期、超出支持范围、缺少城市/时间时都有可操作反馈；修正后无需重填其他字段。

### 6.2 结果首屏（Aha 卡）

固定信息层级，而不是固定文案：

1. **一句身份**：由日主、结构强弱/五行关系等盘面组合生成。
2. **三个盘面事实**：明确标注变量，例如五行分布、透干/藏干、当前运年关系；不得只是关注方向模板。
3. **一个当前行动**：同时读取盘面结构、当前运/年信号、用户关注方向和关系信息，输出低风险小行动。
4. **一个验证问题**：邀请用户用现实经历校验，而不是要求相信。
5. **一个不确定性**：时间边界、城市未确认、算法近似等真实风险。

验收：以至少 12 组覆盖不同日主、强弱、运年和关注方向的样例进行快照比对；核心行动不能只因“事业/关系/财富”选项不同而套用相同句式。

### 6.3 长篇解读

- 默认展开“先看摘要”，其余章节折叠。
- 章节标题展示一句结论和预计阅读量；展开后再呈现依据、解释、行动。
- “盘面事实”“解释假设”“行动建议”使用稳定的视觉标签。
- 健康、投资、法律、关系决策旁显示适当边界，不放置恐惧型 CTA。

验收：在 360px 设备上，用户从结果顶部到任一章节不需要滚过整篇内容；折叠状态可由键盘控制并正确读屏。

### 6.4 分享

- 分享前先展示实际预览与“将包含哪些信息”。
- 默认仅包含昵称/自定义称呼（可选）、身份句、行动、人物相似度、站点品牌和运行时 URL。
- 完整生日、出生时刻、出生地、四柱明文默认均不出现；用户即使主动选择展示，也必须二次确认。
- 使用 `location.href`/站点当前来源构建回流地址，不写死本地端口或未知生产域名。
- Canvas 海报必须有文本 CTA；二维码只有在生成库可靠、离线可用且测试通过后加入。

验收：检查原生分享、复制文本和下载海报三种产物；三者默认脱敏、含可访问站点地址，且在生产域名下不出现 `localhost`。

### 6.5 视觉与无障碍规范

- 视觉关键词：克制、清晰、可信、有东方文化质感但不玄虚。
- 正文与背景对比度至少满足 WCAG AA；焦点样式不可被移除。
- 点击目标建议不小于 44×44 CSS px；移动端不使用只靠 hover 的信息。
- 动画尊重 `prefers-reduced-motion`；装饰不阻挡指针事件。
- 弹层出现时管理焦点，关闭后焦点回到触发元素；Esc 可关闭非强制弹层。

验收：完成键盘-only 主流程、200% 字体缩放、减少动画模式以及 320px 最窄视口检查；页面无横向滚动条。

## 7. 技术架构与质量策略

### 7.1 静态架构

```text
Cloudflare Pages（静态托管 + CDN）
        │
        ├─ index.html / privacy.html / methodology.html
        ├─ css：设计令牌、布局、组件、响应式、无障碍状态
        └─ js
           ├─ engine / engine-plus：历法与命盘计算
           ├─ interpret：解释规则
           ├─ product：行动、稳定性、分享等产品层
           ├─ figures：本地人物库
           └─ app：状态、视图、事件与本地档案

浏览器本地边界
├─ localStorage：档案、最近使用、非敏感偏好
├─ Canvas：本地海报
└─ JSON：用户主动导入/导出
```

架构验收：断网后已加载页面的核心计算、档案和海报不依赖远程 API；Network 面板不发送出生信息或命盘结果。

### 7.2 启动韧性

- 开场遮罩默认不阻塞页面；JavaScript 增强成功后再进入需要脚本的状态。
- 初始化按依赖边界包裹 `try/catch`，错误写入可理解的恢复面板，而不是白屏。
- 恢复面板提供“刷新”“返回输入”“导出已有档案（若可用）”等安全动作。
- 非关键模块（人物库、海报）失败不得阻止基础排盘；核心引擎失败则禁用计算并解释原因。

验收：分别模拟缺失人物库脚本、海报异常和核心引擎异常；前两者基础排盘仍可用，后者显示明确错误与恢复路径，均无永久遮罩。

### 7.3 数据与导入安全

- 导入前校验文件扩展名/MIME、大小、JSON 结构、应用标识、版本、字段类型与记录数量上限。
- 解析和验证在临时对象完成；全部通过后才合并，异常时不覆盖原档案。
- 对用户可见地说明 localStorage 是设备浏览器内的明文数据，清缓存/无痕模式/换设备可能丢失。
- 导出内容标注版本和时间，未来迁移采用明确的 schema 版本函数。

验收：空文件、非 JSON、超限文件、超量记录、未知版本、恶意 HTML 字符串和缺字段样例均被安全拒绝；原数据保持不变。

### 7.4 静态安全基线

通过 Cloudflare Pages `_headers` 配置至少包含：

- `Content-Security-Policy`：只允许必要的同源脚本/样式/图片；若存在合法内联样式或 Cloudflare Insights，需精确适配并记录原因。
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy`
- `Permissions-Policy`：默认关闭不需要的摄像头、麦克风、地理位置等能力。
- 合理的 frame 限制（CSP `frame-ancestors`）和静态资源缓存策略。

验收：生产环境响应头实际可见；浏览器控制台没有 CSP 阻断核心功能；Canvas data/blob URL、下载与可选 Analytics 在策略中按最小权限工作。

### 7.5 算法可信度

- 将“排盘可信度”改为更准确的“柱位稳定度”或“时间确定度”，避免把规则稳定误解为科学准确率。
- 方法页公开支持日期范围、节气算法近似、真太阳时计算、子时流派差异与已知边界。
- 建立权威万年历/节气数据的离线交叉验证样本，重点覆盖节气切换、23 点换日、真太阳时跨日、闰月和日期范围边界。

验收：关键边界样本有来源、预期值和测试结果；不把测试覆盖率描述成命理有效性的证明。

### 7.6 自动化测试金字塔

| 层级 | 覆盖 | 发布门槛 |
|---|---|---|
| 回归测试 | 历法、四柱、边界算法 | `test/regression.js` 全部通过（当前基线 56/56） |
| 冒烟测试 | 人物库、产品层 | `figures-smoke.js` 与 `product-smoke.js` 全部通过（当前基线 60 人、9/9） |
| 浏览器 E2E | 开场、有效/无效输入、排盘、标签、保存/恢复、海报、错误监控 | Chromium 桌面端和 360px 至少各跑一遍，0 个未处理异常 |
| 视觉/响应式 | 320/360/390px 与桌面截图 | 无遮挡、无横向溢出、关键 CTA 可见可点 |
| 故障注入 | 缺失脚本、localStorage 异常、损坏导入 | 有降级/恢复反馈，不丢已有数据 |
| CI | GitHub Actions 执行全部检查 | 任一必需检查失败则不可视为可发布版本 |

E2E 必须使用真实本地静态服务器模拟生产路径；启动端口仅用于开发/测试，不代表 Cloudflare 部署需要常驻服务。Cloudflare Pages 仍直接托管静态文件。

## 8. 增长引擎

### 8.1 三个传播钩子

1. **真太阳时前后，我换柱了吗？**：用可验证的规则差异制造认知惊喜。
2. **我最像哪位历史人物？**：用身份表达降低分享门槛。
3. **今年只做对一件事**：用低风险行动卡创造收藏和回访价值。

所有钩子都必须回到同一个闭环：

```text
脱敏卡片 → 运行时站点地址 → 快速排盘 → 30 秒 Aha
   ↑                                      ↓
再次分享 ← 月度反思 / 当期行动 ← 本地保存
```

验收：任何传播素材都能回站、默认不泄露生辰；新访客从落地到排盘无需先注册或付费。

### 8.2 AARRR 指标体系

在没有可验证 Analytics 前，以下均为“定义”，不是已取得的数据。若开启 Cloudflare Web Analytics 或未来匿名事件系统，事件属性不得含个人命盘信息。

| 阶段 | 核心事件 | 指标定义 | 第一阶段目标假设 | 验收标准 |
|---|---|---|---|---|
| Acquisition 获取 | 到达首页 | 独立访问、来源渠道、落地页 | 先建立可重复的 3 类内容入口 | UTM/来源中不包含生辰；渠道可区分 |
| Activation 激活 | 首次成功排盘并看到 Aha 卡 | 成功排盘人数 / 有意图访问人数 | ≥35%（待真实数据校准） | 排盘成功事件只记录布尔值/耗时段，不记录输入和四柱 |
| Retention 留存 | 7/30 天回访或恢复最近档案 | 回访用户 / 已激活用户 | 7 日 ≥12%，30 日 ≥6%（假设） | 有本地恢复与月度回看入口；口径可复现 |
| Referral 推荐 | 成功触发分享且接收者回站 | 分享人数、分享回站访问、回站激活 | 分享触发 ≥8%，回站激活 ≥20%（假设） | 分享 URL 可归因但不包含用户命盘标识 |
| Revenue 收入 | 深度内容购买意向/真实订单 | 意向率、购买率、退款率、客单价 | 先验证意向，不承诺收入 | 无支付前只记录明确自愿意向，不能称为订单/付费用户 |

### 8.3 内容与冷启动

- 内容支柱 A：规则科普——真太阳时、节气定月、子时流派的可视化对比。
- 内容支柱 B：行动卡——“某种盘面信号如何转成现实实验”，强调可验证与非决定论。
- 内容支柱 C：趣味身份——历史人物相似度，但明确“趣味对照”而非人格鉴定。
- 种子用户：传统文化学习者、重视隐私的自我探索者、半专业排盘用户。
- 冷启动方式：10–20 人可用性访谈、真实任务录屏、规则边界共创；不购买模糊流量冲虚荣指标。

验收：首月完成至少 10 个结构化访谈和 5 次可观察可用性测试；每条内容只设一个回站目标；用户反馈能映射到路线图条目。

## 9. 商业模式与单位经济框架

### 9.1 免费边界

以下能力长期保持免费，因为它们决定产品可信度和用户数据自主权：

- 基础排盘和关键时间规则；
- 柱位稳定度/时间确定度与方法说明；
- 基础白话摘要和风险边界；
- 本地档案的查看、导出和恢复；
- 默认脱敏分享。

验收：用户不付费也能完成可信排盘、理解核心结果、带走自己的数据；不存在“付费才能修正排盘错误”或“恐惧后解锁化解”的设计。

### 9.2 商业假设（先验证，后开发）

| 假设产品 | 价值单位 | 静态阶段可做 | 正式售卖前置条件 | 验收标准 |
|---|---|---|---|---|
| 深度主题报告 | 一次性交付某个事业/关系/能力主题 | 展示目录、匿名样页、意向按钮/访谈 | 支付、订单、授权下载、退款与客服 | 至少 20 个目标用户评价样品，确认明确付费理由 |
| 年度行动报告 | 一年节奏、关键月份、复盘模板 | 免费样章与本地体验版 | 同上，且内容经过边界审查 | 意向不是由灾祸恐吓产生；用户能说明实际用途 |
| 专业多档案工具 | 更高效的档案、批量对照、品牌报告 | 原型和专业用户访谈 | 授权系统、商业条款、数据保护 | 至少 10 名专业/半专业用户验证高频任务和节省时间 |
| 可选加密同步 | 跨设备、可恢复、端到端加密 | 仅解释需求，不伪实现 | 后端、密钥模型、安全审计、数据删除与合规流程 | 威胁模型通过评审；用户可导出和彻底删除数据 |

价格只能在访谈/测试中作为假设。优先测试透明的一次性购买，不默认自动续费；任何订阅都必须在持续价值、取消流程和客服能力被验证后再考虑。

### 9.3 单位经济模型

正式投入商业化前记录以下变量：

- `激活成本 = 获客支出 / 首次成功排盘用户数`
- `CAC = 获客与销售总支出 / 新付费用户数`
- `毛利 = 实收收入 - 支付通道费 - 内容/算力/客服等可变成本`
- `单客毛利 LTV = 平均客单价 × 平均购买次数 × 毛利率 - 单客退款与服务成本`
- `回收期 = CAC / 月均单客毛利`
- `意向转化率 = 有效购买意向人数 / 看过完整样品的人数`
- `退款率 = 退款订单数 / 支付订单数`

商业放量门槛建议：有真实支付后再计算 LTV/CAC；连续两个观察周期满足 `LTV/CAC ≥ 3`、回收期可接受、退款与投诉在预设阈值内，才能扩大付费获客。未产生订单时不得用“意向”代替收入。

验收：财务表中的每个数都能追溯到订单、渠道或成本凭证；免费自然流量与付费流量分开计算；不报告无法验证的行业平均值作为自有成绩。

## 10. 30 / 60 / 90 天 GTM

### 0–30 天：把产品做成“敢公开发”的版本

目标：修通主链、建立信任和质量门，拿到第一批真实使用证据。

- 完成 P0：快速输入、Aha 摘要、折叠阅读、未排盘状态、分享回流、最近档案。
- 上线隐私页、方法页、静态安全头和导入保护。
- 建立桌面/移动真实浏览器 E2E 与 CI。
- 访谈 10–20 位种子用户，观察 5 位用户从零完成排盘。
- 发布 6–9 条内容，平均覆盖三个传播钩子，不做付费投放承诺。

验收：P0 验收项全部通过；严重可用性问题为 0；有结构化访谈记录；能得到首批激活、分享和回访基线，而不是只看访问量。

### 31–60 天：证明“为什么会回来、为什么会分享”

目标：形成留存和传播的可验证信号。

- 实现月度行动/反思、本地历史对照、更多脱敏卡片样式。
- 优化真太阳时换柱故事和规则差异展示。
- 对首页价值表达、Aha 信息顺序做不含敏感数据的轻量实验。
- 与学习型/专业型小社群开展 2–3 次规则共创或测评。
- 制作深度主题报告样品，只测需求与价格接受度，不伪接支付。

验收：至少一个传播钩子的“分享 → 回站 → 激活”链路可测；7 日回访有基线；20 位目标用户完成样品价值/价格访谈。

### 61–90 天：选择一个被数据证明的增长面

目标：集中资源，而不是同时扩张所有方向。

- 根据激活、留存、分享与访谈数据，选择“规则可信”“行动复盘”或“趣味身份”中的一个主增长面。
- 完成专业档案效率原型或年度行动报告 MVP 样品，二选一。
- 评估是否值得引入后端/支付：安全、合规、客服、成本和转化必须共同过门槛。
- 建立季度内容计划、版本节奏和用户反馈看板。

验收：选择依据有数据和访谈证据；被放弃/延后的方向有明确原因；若后端条件未满足，继续保持纯静态，不用前端模拟交易。

| P0-11 Cloudflare 安全基线 | `_headers`、CSP、nosniff、Referrer/Permissions Policy、缓存策略 | 减少静态站常见攻击面 | 线上响应头实测存在；核心脚本、Canvas 和下载无 CSP 错误 |

## 12. 发布验收与交叉评审

每个公开版本必须通过以下用户模拟：

1. **新用户**：不看说明，从首页完成输入并理解 Aha 卡。
2. **回访用户**：从最近档案恢复，找到上次行动并生成分享卡。
3. **错误用户**：输入非法/边界日期、未确认城市、损坏导入文件，能自行恢复。
4. **移动用户**：320/360/390px 下无遮挡、横向溢出和不可点控件。
5. **键盘/辅助技术用户**：能走完主链，焦点顺序、标签、弹层状态正确。
6. **隐私审计者**：检查 Network、分享文本、海报、导出和埋点，确认默认保护。
7. **故障场景**：移除一个非核心脚本以及一个核心脚本，验证降级与恢复。

六角色交叉评审的“一票否决”项：

| 角色 | 一票否决条件 |
|---|---|
| 市场 | 定位退回“又一个 AI 算命/排盘大全”，或宣传不可验证的准确率 |
| 产品 | 首次价值必须阅读长文或注册后才能获得，核心链路存在死路 |
| 体验 | 关键按钮被遮挡/不可点，移动端横向溢出，错误无修复路径 |
| 技术 | 自动化测试失败、脚本异常白屏、导入能覆盖/污染已有档案 |
| 增长 | 默认分享泄露生辰、分享没有回站地址、用暗黑模式诱导传播 |
| 商业 | 前端伪会员、恐惧付费、基础排盘纠错或用户数据恢复被收费 |

最终 Definition of Done：所有 P0 条目有代码或页面证据；现有回归/冒烟和 E2E 全绿；桌面/移动截图人工复核；生产静态部署可访问；Git 提交可追溯；隐私、方法、分享和失败恢复均完成专项检查。

## 13. 决策看板

每两周只回答以下问题，避免继续用功能数量替代产品进步：

1. 用户是否更快看到第一条有用结论？
2. 用户能否说明该结论来自什么盘面依据？
3. 用户为什么保存、为什么回来？
4. 哪一种卡片让接收者真正回站并完成排盘？
5. 哪一类人愿意为什么具体结果付费，而不是泛泛表示“有兴趣”？
6. 本轮是否出现隐私、焦虑营销、算法边界或浏览器兼容性风险？

优先级调整规则：先修阻断使用与信任的问题，再提升激活；激活成立后再优化留存和传播；有稳定留存/传播证据后才扩大商业化投入。任何新增功能必须绑定一个 JTBD、一个可观察指标和一个明确验收标准，否则不进入开发。

## 14. 结论

天枢的爆款机会不在“算得更玄”，而在“让用户看得懂、查得到、做得到、敢保存、敢分享”。纯静态和本地优先不是权宜之计，而是现阶段最可信的差异化资产：它迫使产品把核心价值真正做进算法、交互和内容，而不是依赖登录墙、云端话术和焦虑付费。

执行顺序必须保持清晰：

> **先通过 P0 把可用与可信做成硬质量门，再用 P1 证明回访，用 P2 证明传播/专业价值，最后才决定 P3 是否值得引入后端与交易。**

只要所有团队决策持续围绕这一顺序，天枢就能从“功能完整的排盘页”升级为“用户愿意长期保留的个人节奏工具”。
