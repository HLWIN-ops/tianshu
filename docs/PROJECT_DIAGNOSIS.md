# 天枢《项目诊断报告》

> 阶段：1 / 5
> 日期：2026-07-16
> 诊断对象：Cloudflare Pages 免费版纯静态前端

## 一、总判断

天枢不是一个从零开始的 Demo。它已经拥有专业排盘 MVP 的主要骨架：公历/农历、精确时分/时辰、349 个离线城市、真太阳时、两种子时规则、四柱十神藏干神煞五行、大运流年、长篇解读、本地档案、人物匹配和 Canvas 海报。

但改造前它更像“功能完整的排盘工具”，还不像“用户能快速理解、愿意保存、愿意分享、愿意回来的消费产品”。综合爆款准备度约 **4.9 / 10**：

| 维度 | 诊断 | 成熟度 |
|---|---|---:|
| 技术 MVP | 核心本地计算和主要结果链路已具备 | 约 75% |
| 消费产品 | 有内容，没有足够快的 Aha 与信息分层 | 约 55% |
| 增长产品 | 有海报动作，没有稳定回流和复访机制 | 低于 30% |
| 商业产品 | 没有支付、权益、归因和单位经济数据 | 约 10% |

核心诊断：

> **产品已经“算得出来”，但还没有把“为什么可信、此刻做什么、为什么回来、如何安全分享”组织成一个闭环。**

## 二、现有优势与可复用资产

1. **隐私和成本结构天然有利**：排盘、解读和海报都在浏览器完成，不依赖远程 AI、地图 API、账号或数据库，适配 Cloudflare Pages 免费静态托管。
2. **专业口径比普通免费工具完整**：时间精度、城市经度、真太阳时、时区和子时规则已进入输入链路。
3. **结果能力广**：四柱、十神、藏干、神煞、五行、大运、流年和长解读已经形成可用内容资产。
4. **有初步留存基础**：本地档案支持保存、打开、编辑、删除、JSON 导入导出。
5. **有传播素材基础**：人物相似度与 720×1280 Canvas 海报适合社交表达。
6. **工程已有回归资产**：算法回归、人物库冒烟和产品冒烟测试可以继续扩充，而不是从零搭门禁。

## 三、六位专家体检

### Agent 1 — 市场情报官 📡

**判断**：专业排盘功能已经进入红海，AI 长报告也在快速同质化。单纯增加神煞、命理门类或聊天入口，很难建立稳定优势。

**致命问题**：没有一句能被用户复述的差异化定位；“准确、专业、全面”都是竞品共同话术。

**机会**：把“本地隐私、规则可追溯、不确定性透明、行动可验证”组合成一个尚未被头部产品完全占据的位置。

### Agent 2 — 产品架构师 🏗️

**判断**：功能不少，但核心旅程仍是“填长表单 → 看长结果”，没有清晰的 30 秒价值时刻。

**致命问题**：首次输入负担重；结果首屏缺少一个身份句、三条命盘事实和一个当期行动；回访用户仍要先进入档案页找记录。

**优先动作**：快速路径、Aha 速览、最近档案、7 天试做结论、分享回流。

### Agent 3 — 交互体验设计师 🎨

**判断**：传统视觉有辨识度，但改造前存在真实“看得到、点不到”风险；长解读在移动端形成约 8540px 的连续页面。

**致命问题**：开屏按钮的入场动画尚未稳定，开屏就自动退出；未排盘标签表现得像可点击按钮，点击后才弹错；结果信息密度高、章节默认全展开。

**优先动作**：默认不阻塞开屏、结果前锁定标签、字段级错误聚焦、长解读折叠、弹层焦点管理、320/360/390px 验收。

### Agent 4 — 技术负责人 ⚙️

**判断**：纯静态结构简单、成本低，但发布质量主要依赖人工点击。

**致命问题**：一个经典脚本失败可能让全站初始化中断；缺少真实浏览器 E2E、CI、CSP、安全头和导入边界；节气/均时差使用低精度公式，尚缺权威交叉验证。

**优先动作**：启动隔离与降级、可恢复错误面板、Playwright 临时静态服务器、GitHub Actions、`_headers`、导入 schema/版本/大小/数量校验。

### Agent 5 — 增长黑客 📈

**判断**：人物匹配和海报具备传播潜力，但改造前没有稳定产品 URL、CTA 或隐私默认值，分享并不等于带来新用户。

**致命问题**：分享文本和海报没有完整回站链路；缺少最近档案与周期任务，用户完成一次排盘后没有明确理由回来。

**优先动作**：脱敏结果卡 → 运行时 URL → 快速排盘 → 保存本地 → 月度验证 → 再分享。

### Agent 6 — 商业模式设计师 💰

**判断**：当前是“卖点明确、交付完整的免费工具”，还不是“可收费、可度量、可持续经营的产品”。商业成熟度约 34 / 100。

**致命问题**：没有账号、订单、支付、服务端权益或漏斗数据；全部能力都在前端，任何“会员开关”都可绕过；当前无法计算 LTV/CAC。

**原则**：基础排盘、稳定度、隐私和数据恢复继续免费。先用访谈与完整样品验证深度主题报告、年度行动报告、专业多档案或可选加密同步的意愿；没有后端条件时不伪造会员。

## 四、致命问题排序

### P0 — 不修就不应公开推广

1. 开屏存在可复现的可点性竞态，真实用户会遇到“按钮点不开”。
2. 首次输入长，核心价值出现太晚。
3. 行动建议过度依赖关注主题模板，缺少盘面/大运/流年差异。
4. 分享默认包含过多命盘信息，且缺少运行时回站地址。
5. 长解读移动端过长，用户难以找到重点。
6. 未排盘标签是死路式交互，没有明确锁定状态。
7. 没有浏览器级质量门，单元测试抓不到遮罩、焦点和布局问题。
8. 本地档案明文且可能丢失，但缺少独立隐私说明与方法边界。
9. JSON 导入缺少文件大小、MIME、schema 和版本边界。
10. 缺少 CSP、Permissions Policy、Referrer Policy 等静态安全基线。

### P1 — 决定是否会复访

- 最近档案一键恢复。
- 7 天试做、现实证据与保留/调整/放弃结论的本地记录。
- 真太阳时前后是否换柱的差异卡。
- 12 组以上个性化样例矩阵。
- 键盘、200% 字体、减少动画专项。

### P2 — 决定能否形成增长资产

- 多模板脱敏分享卡与可验证回流。
- 方法知识库、算法版本与边界案例。
- 专业档案搜索、排序、冲突预览。
- 有真实数据后再接匿名漏斗测量。

### P3 — 只有条件满足才启动

- 正式一次性购买。
- 端到端加密同步。
- 可选远程 AI 深读。
- 专业商业版。

## 五、阶段 1 结论

天枢值得继续改造，因为它已有低成本、隐私友好且相对完整的本地技术资产。最优路径不是复制头部竞品的功能数量，而是先把 **可用、可信、可解释、可回访、可脱敏分享** 做成硬门槛，再用真实用户和数据决定商业化。
